1Problem

Given n items with weights wi and values vi, and a knapsack of capacity C — pick a subset that maximizes total value without exceeding capacity. 

maximize   Σ vᵢ · xᵢ
subject to Σ wᵢ · xᵢ ≤ C
           xᵢ ∈ {0, 1}

NP-hard. Same family as driver-to-region assignment, truck loading, and the integer programs PuLP/CBC solves under the hood — studying Branch & Bound here means studying the production engine.

2Approach

  1. Branch. DFS over include/exclude decisions, ordered by value/weight ratio so the greedy guess is tried first.
  2. Bound. LP relaxation upper bound — a single O(n) greedy sweep.
  3. Prune. Skip any subtree whose bound ≤ current incumbent.
Same algorithm in three languages — Python, C++, x86-64 NASM (Win64 ABI, pure integer math, no FPU). The language is the only variable. 
void branch(int level, int cw, int cv) {
  if (cv > best) best = cv;
  if (level == N) return;
  if (upper_bound(level, cw, cv) <= best) return;  // PRUNE
  if (cw + W[level] <= CAP)                         // INCLUDE
    branch(level+1, cw + W[level], cv + V[level]);
  branch(level+1, cw, cv);                           // EXCLUDE
}

8Verification

Four solvers, four angles — 9 pytest checks, all passing in < 0.25 s.

  • Known answers: PLAN-v1 (value = 11) and disaster relief (value = 598).
  • Cross-solver agreement: PuLP, B&B, brute force all match on the canonical instance.
  • Random-instance fuzzing: 50 seeded instances (n ∈ [1, 12]) — B&B must equal brute force on every one. Brute force is the oracle.
